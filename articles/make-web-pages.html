¿Cómo se hacen las páginas web?

Por: Irving Mendoza Martinez

Comencemos . . .

Todos hemos entrado alguna vez a una página web o puede que incluso las usemos en nuestra vida cotidiana, ya sea por el trabajo, la escuela o el ocio. Son bastante útiles debido a la información que algunas nos pueden proporcionar podemos informarnos un determinado tema, usar las herramientas que nos brindan o entretenernos.

Sin embargo, sabemos muy poco con respecto a su funcionamiento o del proceso que hay para crearlas, subirlas a internet y los conocimientos sobre tecnologías y lenguajes para hacerlo.

En este texto se tratara de abarcar el como y lo que conlleva crear una página de internet profundizando sólo lo suficiente como para tener una idea clara del proceso y del porqué de ciertas cosas, a la vez que se hace de manera comprensible para cualquiera y así poder conocer más sobre el funcionamiento de lo que usamos. Podría ser servirte si te interesa el desarrollo web o bien puede simplemente ser algo interesante al tener la curiosidad de conocer y descubrir algo nuevo.

¿Qué es realmente una página web?

Una página web básicamente es un documento, específicamente un documento HTML, en dicho documento se escribe código HTML (HyperText Markup Language), el cual se usa para representar elementos que estructuran la información de la página, por ejemplo: textos, hipervínculos (links), imágenes, vídeos, tablas, formularios, contenedores de información, etc.

El documento puede ser subido a internet y así formar parte de la World Wide Web o simplemente web, que es el nombre con el que se conoce al conjunto de páginas subidas a internet que existen. En resumen una página web es un documento HTML que está en la web.

Este tipo de documento puede ser leído por un navegador web, existen varios de éstos: Google Chrome, Mozilla Firefox, Microsoft Edge, Opera, Safari y el recientemente popularizado Brave, entre otros.

Todos estos son programas que al leer el código HTML pueden mostrarnos la información con el formato indicado en dicho código, ósea mostrarnos la página web. En el código está toda la información, la cual debe estar estructurada de manera que el navegador pueda saber que es cada cosa: títulos, párrafos, enlistados, cabeceras, contenido principal, contenido secundario, pie de página, etc.

Junto a esto, los navegadores también permiten localizar una página específica entre la web, esto por medio de una dirección llamada URL, que es una dirección única para cada página.

Lenguaje HTML

Como ya se mencionó, el documento HTML debe contener código en lenguaje HTML, en éste código, principalmente, se marca toda la estructura de la página. Las reglas y patrones para escribir este código (sintaxis) se basan en etiquetas las cuales son la representación de los elementos de la página.

Algunas de las etiquetas más importantes para representar elementos son:

  • Etiquetas h: titulos
  • Etiqueta p: parrafos
  • Etiqueta a: hipervinculos
  • Etiqueta img: imagenes
  • Etiqueta div: contenedor para agrupar contenido

Existe una enorme variedad de etiquetas en HTML que cumplen diversas funciones en la estructuración de la página y de su contenido. Toda cosa que puede ser representada por una etiqueta HTML dentro del viewport se conoce como elemento HTML.

Para ver las muchas otras etiquetas que existen, se pude ver la documentacion de la MDN: Referencia de Elementos HTML | MDN

Nota: se puede ver el código HTML de cualquier página web. Esto se hace presionando F12 y después haciendo click en el inspector, para salir se vuelve a presionar la tecla.

De esta manera es básicamente como se escribe el código HTML de una página. Pero una página no solo es información, imágenes y links; También tiene que verse estéticamente bien y ahí es donde entra otro lenguaje llamado CSS.

Aspecto visual con CSS

Después de definir la estructura de la página con HTML, se necesita una forma de poder establecer colores, fuentes de letra, crear animaciones, poder mover y posicionar la información, etc. Esto se hace con el lenguaje CSS (Cascade Style Sheets), un lenguaje para modificar las propiedades por defecto de los elementos HTML.

Algunas de las propiedades que tienen los elementos html son:

  • color - El color del texto del elemento.
    Ejemplo: color: #0000FF; (azul)
  • background-color - el color del fondo del elemento.
    Ejemplo: background-color: #000000; (negro)
  • width - El tamaño o el ancho de un elemento.
    Ejemplo: width: 40px; (40 pixeles)
  • height - El largo del elemento.
    Ejemplo: height: 40px;
  • font-size - Tamaño de letra.
    Ejemplo: font-size: 16px;
  • font-family - Tipo de fuente.
    Ejemplo: font-family: sans-serif;

Los elementos HTML tienen una infinidad de propiedades y todas ellas pueden modificarse con CSS, esto nos permite poder darle un buen aspecto visual a la página.

El código CSS se puede escribir directamente en el mismo documento pero es más recomendable hacerlo en un archivo por separado y posteriormente enlazarlo con el documento.

Comportamiento, funcionalidades y otras cosas con JavaScript

Los dos lenguajes ya anterior mencionados (HTML y CSS), podrían ser ya suficientes para crear una página informativa simple, sin embargo incluso si se trata de la elaboración de una página muy sencilla puede ser necesario algo más, ya que puede existir la necesidad de hacer que la pagina sea mas dinamica. Ahí es donde entra JavaScript que junto a HTML y CSS conforman las tres tecnologías base para crear una página web

Pero, ¿para qué sirve exactamente JavaScript? JavaScript es un lenguaje de programación que fue creado para ser usado junto a HTML y así hacer las páginas más interactivas y dinámicas o para que se cumpla alguna función específica que no se puede hacerse con HTML y CSS.

Algunas de las cosas que permite hacer JavaScript son las siguientes:

  • Actualizar el contenido de una página sin necesidad de tocar el código HTML.
  • Hacer animaciones.
  • Verificar los datos de un formulario.
  • Casi cualquier cosa . . .

Como ya se mencionó, JavaScript es un lenguaje de programación, debido a esto no se explicara a detalle el cómo usarlo ni escribir su código, pues eso sería tema para otra ocasión, ya que explicar y extenderse en los conceptos de la programación podría hacer salir del enfoque principal al lector.

Para ver ejemplos y desmostraciones de lo que se puede hacer con JS en una pagina puede ver: Demostraciones del poder de JavaScript

¿Cómo trabaja una página web? | Front-end y Back-end

Una página web funciona con una arquitectura de cliente-servidor.

El servidor es donde se encuentran los archivos que integran la página (este puede ser una computadora muy potente ubicada en algún centro de datos).

El cliente por su parte, es el navegador web en el dispositivo desde donde el usuario visita la página (un smartphone, tablet, laptop o PC).

Cuando se accede a una página, lo que sucede es que el cliente hace una petición http al servidor que corresponda con la dirección de la página y tras procesar dicha petición, éste devuelve una respuesta http, osea los archivos necesarios para cargar la página como el documento html, hoja de estilos, scripts, imágenes, videos, etc.

Y con esto podemos saber en lo que consiste el front-end y el back-end:

Front-end

El front-end es el desarrollo de la parte de la página que se ejecutará del lado del cliente (con la que el usuario interactúa directamente), es decir, los archivos que el servidor envíara al cliente por medio de la respuesta http tras haber sido solicitados y a su vez es todo lo que se ha visto hasta éste punto en éste articulo (HTML,CSS,JavaScript).

Back-end

El back-end, en cambio, es la parte que se encarga de lo que ocurre del lado del servidor, por ejemplo el software que valida la información (usando lenguajes de programación), las bases de datos, la administración de los recursos, etc. Al igual que en JavaScript, no se ahondará mucho en el tema del back-end, se simplificará diciendo que esta es “la parte trasera” de una página web, además es posible hacer una página web sin saber sobre back-end ya que muchos servicios de alojamiento cubren esa parte.

Subir a internet

Una vez que ya se ha desarrollado tanto la parte de front-end y la del back-end el sitio web debe ser alojado en un servidor donde la parte del front-end será almacenada y donde se ejecutará lo desarrollado por el back-end (de nuevo, la parte del back-end puede ser omitida si se trata de algo muy simple ya que hay hostings que se encargan de ésto).

Para subirla se tiene más de una opción, se puede alojar en un hosting (un servidor compartido), existen varias opciones de hosting, algunas son gratuitas y otras de pago. Otra opción también es el almacenamiento en la nube.

Y una vez hecho ésto, se ha concluido el desarrollo de la página. Pero a pesar de eso, el proyecto puede ir escalando según su crecimiento o por el surgimiento de nuevas tecnologías y la obsolescencia de otras.

En conclución

Para crear una página web (al menos para la parte del front-end) se usan tres tecnologías básicas: HTML, CSS y JavaScript.

HTML plasma la información y la semántica de la página en el documento , CSS modifica el aspecto visual por defecto para poder personalizarlo y JavaScript da interactividad y dinamismo a la página, justo como un programa o aplicación.

Aparte se usan tecnologías que corren del lado del servidor como bases de datos y lenguajes de programación para back-end. Cuando la página es terminada ésta es alojada en un servidor para que se pueda acceder a ella desde el navegador de cualquier dispositivo con internet.

Este artículo no pretende ser una guía completa de cómo desarrollar una página web. En la web existe una infinidad información más completa que la presente.
